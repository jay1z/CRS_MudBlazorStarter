# Site Visits

The site visit is where you physically inspect the community's components. This guide covers scheduling, preparation, conducting the inspection, and documenting your findings.

---

## Overview

A thorough site visit is essential for an accurate reserve study.

### Site Visit Goals

| Goal | Description |
|------|-------------|
| **Verify components** | Confirm what exists |
| **Assess condition** | Rate current state |
| **Estimate remaining life** | How long until replacement |
| **Document findings** | Photos and notes |
| **Identify issues** | Problems or concerns |
| **Gather information** | Manufacturer info, install dates |

### Timeline

| Stage | Typical Timeframe |
|-------|-------------------|
| Financial info received | Prerequisite |
| Schedule visit | 1-2 weeks out |
| Conduct visit | 2-8 hours on-site |
| Enter data | 1-3 days after |
| Complete documentation | 1 week after visit |

## Scheduling a Site Visit

### When to Schedule

Schedule after:

- Financial information is **received and accepted**
- Study status is **"SiteVisitPending"** or similar
- You have **reviewed** community information

### How to Schedule

1. Open the **study**
2. Click **"Schedule Site Visit"**
3. Select a **date and time**
4. Add **location details** (address, gate codes)
5. Optionally **invite the client**
6. Save the event

### Using the Calendar

1. Go to **Calendar** in Specialist Tools
2. Click on the **desired date**
3. Select **"Site Visit"** as event type
4. Fill in details:
   - Study/community
   - Start and end time
   - Location
   - Notes

### Client Notification

When you schedule:

- Client can receive **automatic notification**
- Email includes **date, time, and what to expect**
- Client can **confirm** or **request change**

### Confirming Access

Before the visit:

- **Gate codes** - Get access codes
- **Key pickup** - Where to get keys if needed
- **On-site contact** - Who to meet
- **Parking** - Where to park
- **Restrictions** - Pool hours, occupied units, etc.

## Conducting the Inspection

### On Arrival

1. **Meet your contact** (if applicable)
2. **Verify access** to all areas
3. **Review the plan** for the visit
4. **Note weather conditions**

### Inspection Approach

**Systematic Method:**

1. Start at **one point** (e.g., main entrance)
2. Work **systematically** through the property
3. Don't skip areas
4. **Return** to check anything missed

**Category Method:**

1. Inspect all **roofing** first
2. Then all **paving**
3. Then **recreation**
4. Continue by category

### Assessing Components

For each component, evaluate:

| Assessment | Questions |
|------------|-----------|
| **Condition** | What state is it in? (Excellent/Good/Fair/Poor) |
| **Age** | When was it installed? |
| **Remaining Life** | How many years until replacement? |
| **Current Cost** | What would replacement cost today? |
| **Quantity** | How many, what size? |
| **Observations** | Any notable issues? |

### Condition Ratings

Use consistent ratings:

| Rating | Description | Typical Remaining Life |
|--------|-------------|------------------------|
| **Excellent** | Like new, no issues | 80-100% of useful life |
| **Good** | Normal wear, well maintained | 50-80% of useful life |
| **Fair** | Visible wear, needs attention soon | 20-50% of useful life |
| **Poor** | Significant deterioration | 0-20% of useful life |
| **Failed** | Non-functional or unsafe | Needs immediate action |

### Taking Notes

Record key information:

- **Component name** - What it is
- **Location** - Where in the property
- **Quantity** - Count or measurement
- **Condition** - Rating and description
- **Age** - If known or estimated
- **Manufacturer** - If visible
- **Model/specs** - For replacement
- **Issues** - Problems observed
- **Recommendations** - Maintenance or repair

## Taking Photos

### Photo Guidelines

| Guideline | Reason |
|-----------|--------|
| **High resolution** | Detail visibility |
| **Good lighting** | Clear images |
| **Multiple angles** | Complete view |
| **Include context** | Show surroundings |
| **Close-ups** | Defects and details |
| **Label/tag** | Organize by component |

### What to Photograph

| Photo Type | Examples |
|------------|----------|
| **Overview** | Building exterior, pool area |
| **Component** | Roof surface, HVAC unit |
| **Detail** | Serial number, damage |
| **Defect** | Cracks, rust, wear |
| **Measurement** | Tape against surface |
| **Sign** | Manufacturer, model info |

### Photo Organization

Keep photos organized:

- Use **consistent naming** (Component_Location_001.jpg)
- Create **folders** by category
- Add **notes** to photos if possible
- **Backup** immediately

### Photo Quantity

| Component Type | Photos Suggested |
|----------------|------------------|
| Major (roof, pool) | 5-10+ |
| Standard | 2-5 |
| Minor | 1-2 |
| Defects | As many as needed |

## During the Visit

### Time Management

| Property Size | Typical Duration |
|---------------|------------------|
| Small (< 50 units) | 2-4 hours |
| Medium (50-150 units) | 4-6 hours |
| Large (150+ units) | 6-8+ hours |

### Dealing with Access Issues

| Issue | Solution |
|-------|----------|
| Locked area | Contact client for access |
| Occupied unit | Schedule access time |
| Equipment running | Photograph carefully |
| Weather | Reschedule if unsafe |
| Aggressive animal | Keep distance, note issue |

### Client Interaction

If the client accompanies you:

| Do | Don't |
|----|-------|
| Answer questions | Make promises about findings |
| Explain your process | Rush through inspection |
| Note their concerns | Argue about conditions |
| Be professional | Speculate on costs |

## After the Visit

### Immediate Actions

Within 24-48 hours:

1. **Download photos** to computer
2. **Organize** by component
3. **Review notes** for completeness
4. **Note any questions** for follow-up

### Uploading Photos

1. Go to the **study**
2. Navigate to **Photos** or **Site Visit** tab
3. Click **"Upload Photos"**
4. Select files to upload
5. **Tag photos** with components
6. Add **descriptions** if needed

### Entering Site Visit Data

1. Open the **study**
2. Go to **Components** or **Elements** section
3. For each component:
   - Update **condition**
   - Update **remaining life**
   - Add **observations**
   - Link **photos**

### Completing the Visit

When all data is entered:

1. Click **"Mark Visit Complete"**
2. Status changes to **"SiteVisitCompleted"**
3. Add any **summary notes**
4. Study advances to **data entry** or **funding plan**

## Special Situations

### Multi-Day Visits

For large properties:

- Schedule **multiple days**
- Complete **sections** each day
- Document **progress**
- Sync data between days

### Inaccessible Areas

If you can't access something:

- **Document** why (locked, unsafe, etc.)
- **Estimate** from visible areas
- **Note assumptions** in report
- Ask client to **provide access** if critical

### Weather Issues

If weather is bad:

| Condition | Action |
|-----------|--------|
| Light rain | Continue with caution |
| Heavy rain | Postpone roofing, paving |
| Extreme heat | Take breaks, stay hydrated |
| Snow | Reschedule if needed |
| Wind | Be careful with ladders |

### Discovery of Major Issues

If you find serious problems:

1. **Document thoroughly**
2. **Photograph extensively**
3. **Note safety concerns**
4. **Inform client** (if appropriate)
5. **Include in report** with recommendations

## Troubleshooting

### Common Issues

**Q: I forgot to photograph something.**
A: Schedule a quick return visit, or contact the client for photos if minor.

**Q: My photos didn't save properly.**
A: Check storage space. Use backup method. Return if critical items missed.

**Q: The client wants to change the scope after the visit.**
A: Document the request. This may require an amendment. Consult your admin.

**Q: I found significant issues not in the original scope.**
A: Document everything. Include in report. May trigger amendment process.

**Q: I couldn't complete the visit in one day.**
A: Schedule continuation. Document what was completed. Update client.
