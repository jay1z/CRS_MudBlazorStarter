# Proposals

Creating and sending proposals is a key part of the reserve study workflow. This guide covers the full proposal process from creation to client acceptance.

---

## When to Create a Proposal

Create a proposal after:

1. A **study is assigned** to you by admin
2. You've **reviewed the request** details
3. You understand the **scope** the client needs
4. You're ready to **quote pricing**

### Pre-Proposal Checklist

Before creating a proposal, verify:

- [ ] Community information is complete
- [ ] Contact information is accurate
- [ ] Component list is reasonable
- [ ] Study type is clear (full, update, etc.)
- [ ] Any special requests are noted

---

## Creating a New Proposal

### Starting a Proposal

1. Navigate to the **study** assigned to you
2. Click **"Create Proposal"** or **"New Proposal"**
3. The proposal form opens

### Proposal Form Sections

#### 1. Study Information (Pre-filled)

This section auto-populates from the study:

| Field | Source |
|-------|--------|
| Community Name | From request |
| Address | From request |
| Study Type | From request |
| Contact Name | From request |
| Contact Email | From request |

> **Tip**: Verify this information is correct before proceeding.

#### 2. Scope of Work

Define what's included in the study:

| Field | Description |
|-------|-------------|
| **Description** | Detailed scope narrative |
| **Study Type** | Full Study, Update, etc. |
| **Site Visit** | On-site inspection included |
| **Components** | Number of elements included |
| **Report Format** | PDF, printed, digital |
| **Funding Scenarios** | Which analyses included |

**Scope Template Example:**

```
This proposal includes a comprehensive Level I Reserve Study with 
on-site visual inspection of all major common area components. 
The study will include:
- Physical inspection of [XX] components
- 30-year funding projection
- Three funding scenarios (Threshold, Full, and Baseline)
- Digital PDF report delivery
- Executive summary presentation
```

#### 3. Pricing

Configure the study cost:

| Field | Description |
|-------|-------------|
| **Base Price** | Core study fee |
| **Travel Fee** | If applicable |
| **Expedite Fee** | Rush service |
| **Additional Components** | Extra items beyond standard |
| **Discount** | If offering reduced rate |
| **Total** | Calculated sum |

**Pricing Tips:**

- Use your company's **standard pricing** as a baseline
- Add fees for **travel** if the property is distant
- Consider **complexity** (high-rises, large communities)
- Apply discounts for **repeat clients** or **referrals**

#### 4. Payment Schedule

Select how payments are structured:

| Schedule | Description |
|----------|-------------|
| **Full Payment** | 100% due on invoice |
| **50/50** | 50% deposit, 50% on completion |
| **Thirds** | 33% deposit, 33% site visit, 34% final |
| **Quarters** | 25% at 4 milestones |
| **Custom** | Define your own schedule |

#### 5. Timeline

Set expected completion dates:

| Milestone | Typical Timeframe |
|-----------|-------------------|
| Proposal Acceptance | Client dependent |
| Financial Info Collection | 1-2 weeks |
| Site Visit | 2-4 weeks from acceptance |
| Draft Report | 2-4 weeks from visit |
| Final Report | 1-2 weeks from draft |

> **Note**: Communicate realistic timelines. Under-promising and over-delivering builds trust.

#### 6. Terms and Conditions

Your company's standard terms are typically pre-loaded:

- Cancellation policy
- Payment terms
- Warranty/guarantee
- Liability limitations
- Confidentiality

> **Important**: Don't modify legal terms without approval from your admin.

## Saving and Editing Proposals

### Saving Drafts

Your proposal auto-saves, but you can also:

1. Click **"Save Draft"** to explicitly save
2. Exit and return later
3. Status remains **"ProposalCreated"** until sent

### Editing a Proposal

Before sending:

1. Open the study
2. Go to the **Proposal** tab
3. Click **"Edit"**
4. Make changes
5. Save

### Version History

Some systems track proposal versions:

- Original draft
- Revisions made
- Who made changes
- When changes occurred

---

## Proposal Review Process

If your organization requires admin review:

### Submitting for Review

1. Complete the proposal
2. Click **"Submit for Review"**
3. Status changes to **"ProposalReviewed"** (pending)
4. Admin receives notification

### Admin Feedback

The admin may:

| Action | What Happens |
|--------|--------------|
| **Approve** | You can send to client |
| **Request Changes** | Edit and resubmit |
| **Reject** | Discuss with admin |

### Addressing Feedback

1. Open the proposal
2. View **admin comments**
3. Make requested changes
4. Resubmit for review

> **Tip**: If you disagree with feedback, discuss with your admin rather than ignoring it.

---

## Sending the Proposal

### Before Sending

Final checklist:

- [ ] All sections complete
- [ ] Pricing is accurate
- [ ] Timeline is realistic
- [ ] Terms are appropriate
- [ ] Contact email is correct
- [ ] Admin approved (if required)

### Send Process

1. Click **"Send Proposal"** or **"Send to Client"**
2. A confirmation dialog appears
3. Preview the **email that will be sent**
4. Add a **personal message** (optional)
5. Click **"Send"**

### What the Client Receives

The HOA contact receives:

- **Email notification** with summary
- **Link** to view full proposal
- **Instructions** for acceptance
- Your **contact information**

### After Sending

- Status changes to **"ProposalSent"**
- The study moves on the Kanban board
- A **sent date** is recorded
- You can **track** when they view it

## Tracking Proposal Status

### Status Indicators

| Status | Meaning |
|--------|---------|
| **ProposalCreated** | Draft, not sent |
| **ProposalReviewed** | Admin reviewed |
| **ProposalApproved** | Ready to send |
| **ProposalSent** | Delivered to client |
| **ProposalAccepted** | Client signed ✅ |
| **Rejected** | Client declined ❌ |

### Tracking Views

Some systems show:

- When the client **opened** the email
- When they **viewed** the proposal
- How many **times** they viewed
- **Time spent** reviewing

### Following Up

If no response after several days:

1. Check **view tracking** (if available)
2. **Send a follow-up** email
3. **Call** the contact if needed
4. Note follow-ups in **study notes**

---

## Handling Acceptance

### When Client Accepts

You'll receive notification:

- **Email** - "Proposal Accepted"
- **Dashboard** - Notification badge
- **Status** - Changes to "ProposalAccepted"

### What Happens Automatically

Upon acceptance:

1. E-signature is **recorded**
2. Signed copy is **generated**
3. Client receives **confirmation**
4. Invoice may be **created** (if auto-invoicing)
5. Study advances to **next stage**

### Your Next Action

After acceptance:

1. **Request financial information** from the client
2. **Schedule the site visit**
3. Begin **data collection**

---

## Handling Rejection

### If Client Declines

When a proposal is rejected:

1. Status changes to **"Rejected"** or similar
2. You receive **notification**
3. The study may be **cancelled** or **revised**

### Common Rejection Reasons

| Reason | Possible Response |
|--------|-------------------|
| **Price too high** | Discuss scope reduction |
| **Timeline too long** | Offer expedited option |
| **Chose competitor** | Note for future reference |
| **Budget constraints** | Offer payment plan |
| **Changed mind** | Close study |

### Revising After Rejection

If the client wants revisions:

1. **Discuss their concerns**
2. Create a **new proposal** or **revise existing**
3. **Resend** for consideration
4. Document the **negotiation**

## Best Practices

### Writing Effective Scope

- Be **specific** about what's included
- List **exclusions** to avoid misunderstandings
- Use **client-friendly language**
- Match scope to **client's request**

### Pricing Strategies

| Strategy | When to Use |
|----------|-------------|
| Standard pricing | Most situations |
| Premium pricing | Complex properties, rush jobs |
| Competitive pricing | New clients, competitive situations |
| Volume discount | Multiple studies for same client |

### Communication Tips

- **Respond quickly** to questions
- Be **transparent** about limitations
- Set **realistic expectations**
- **Follow up** professionally

## Troubleshooting

### Common Issues

**Q: I can't send the proposal.**
A: Ensure all required fields are complete. Check if admin approval is required.

**Q: The client says they didn't receive the proposal.**
A: Check spam folder. Verify email address. Resend if needed.

**Q: I need to change a proposal after sending.**
A: Contact admin. You may need to void and resend.

**Q: The pricing calculator isn't working.**
A: Check for required fields. Refresh the page. Contact support.
