# Workflow Management

As an admin, you oversee the entire reserve study workflow—from approving requests to publishing final reports. This guide covers your key workflow responsibilities.

---

## Workflow Overview

The reserve study lifecycle flows through these major phases:

```
Request → Approval → Assignment → Proposal → Data Collection
                                                    ↓
Site Visit → Report Generation → QA Review → Publishing → Complete
```

### Your Role in the Workflow

| Phase | Your Responsibility |
|-------|---------------------|
| **Request** | Review and approve |
| **Assignment** | Assign specialist |
| **Proposal** | Review (if required) |
| **Report** | QA review and approve |
| **Publishing** | Publish to client |

---

## Request Queue Management

### Viewing Incoming Requests

1. Check your **dashboard** for pending requests
2. Or go to **Reserve Studies** and filter by **"RequestCreated"**
3. New requests appear with notification badges

### Request Notification

When a client submits a request:

- 📧 **Email notification** sent to admins
- 🔔 **In-app notification** appears
- 📊 **Dashboard counter** updates

### Reviewing Request Details

Click on a request to see:

| Section | Information |
|---------|-------------|
| **Community Info** | Name, address, property type, units |
| **Contact Info** | Primary and secondary contacts |
| **Components** | Requested elements to include |
| **Notes** | Special requests or concerns |
| **Documents** | Any attached files |

### Request Actions

| Action | When to Use | Result |
|--------|-------------|--------|
| **Approve** | Request is complete and valid | Ready for assignment |
| **Request Info** | Need clarification | Client notified |
| **Reject** | Cannot service | Request closed |
| **Edit** | Fix minor issues | Update details |

### Approving a Request

1. Review all sections
2. Click **"Approve"** or **"Approve Request"**
3. Add optional notes
4. Confirm approval
5. Status changes to **"RequestApproved"**

### Requesting More Information

1. Click **"Request Info"** or **"Need More Information"**
2. Specify what's needed
3. Client receives notification
4. Status changes to show pending info
5. Review again when they respond

## Assigning Specialists

### When to Assign

After approving a request, assign a specialist to handle the study.

### Viewing Available Specialists

The assignment dialog shows:

| Information | Description |
|-------------|-------------|
| **Specialist Name** | Available staff |
| **Current Workload** | Number of active studies |
| **Expertise** | Specializations (if tracked) |
| **Availability** | Calendar status |

### Making an Assignment

1. Open the approved study
2. Click **"Assign Specialist"**
3. Select a specialist from the list
4. Add optional notes
5. Click **"Assign"**

### Assignment Notification

The assigned specialist receives:

- 📧 **Email** with study details
- 🔔 **In-app notification**
- Study appears in **their dashboard**

### Workload Balancing

Consider when assigning:

| Factor | Consideration |
|--------|---------------|
| **Current load** | How many active studies? |
| **Complexity** | Large property needs experienced specialist |
| **Location** | Minimize travel time |
| **Expertise** | Match skills to property type |
| **Deadlines** | Who has capacity? |

### Reassigning Studies

To change the assigned specialist:

1. Open the study
2. Click **"Reassign"** or **"Change Specialist"**
3. Select new specialist
4. Add reason for change
5. Both specialists are notified

## Monitoring Study Progress

### Status Dashboard

View all studies organized by status:

| Status Group | Contains |
|--------------|----------|
| **Pending Approval** | New requests awaiting you |
| **Assigned** | Ready for proposal |
| **In Progress** | Active work |
| **Awaiting Client** | Waiting for HOA action |
| **QA Review** | Reports awaiting your review |
| **Complete** | Finished studies |

### Identifying Bottlenecks

Look for studies that are:

| Issue | Indicator | Action |
|-------|-----------|--------|
| **Stalled** | No activity in 2+ weeks | Follow up with specialist |
| **Awaiting client** | Extended client delay | Contact client |
| **Overdue** | Past expected completion | Prioritize or reassign |
| **Stuck in status** | Won't advance | Investigate blocker |

### Overdue Alerts

The system flags:

- Studies **past their target date**
- Invoices **past due**
- Requests **awaiting approval** too long
- Reports **awaiting QA** too long

### Study Aging Report

View how long studies have been in each status:

1. Go to **Reports** or **Aging**
2. Filter by status
3. Sort by **days in stage**
4. Identify and address outliers

## Proposal Review (If Enabled)

If your organization requires proposal review before sending:

### Review Queue

1. Check for proposals with status **"ProposalCreated"** or **"ProposalReviewed"**
2. Specialist submits for your review
3. You receive notification

### Reviewing a Proposal

Check:

| Element | What to Verify |
|---------|----------------|
| **Scope** | Matches client request |
| **Pricing** | Follows pricing guidelines |
| **Timeline** | Realistic dates |
| **Terms** | Standard terms used |
| **Quality** | Professional presentation |

### Proposal Actions

| Action | When | Result |
|--------|------|--------|
| **Approve** | Ready to send | Specialist can send to client |
| **Request Changes** | Needs edits | Returned to specialist |
| **Take Over** | You'll handle | Assigned to you |

## QA Review Process

Quality assurance is critical before publishing reports.

### QA Queue

Studies ready for QA have status:

- **"ReportDrafted"**
- **"FinalReviewPending"**

### Accessing QA Review

1. Click QA notification on dashboard
2. Or go to **Reserve Studies** → Filter by QA status
3. Open the study

### What to Review

#### Report Content

| Section | Check For |
|---------|-----------|
| **Cover/Title** | Correct community name, date |
| **Executive Summary** | Accurate summary |
| **Property Description** | Correct details |
| **Component Inventory** | All components included |
| **Condition Assessments** | Reasonable ratings |
| **Funding Analysis** | Calculations correct |
| **Narrative** | Clear, professional writing |
| **Appendices** | Complete tables/charts |

#### Technical Accuracy

| Item | Verify |
|------|--------|
| **Useful life** | Appropriate for components |
| **Costs** | Current and reasonable |
| **Calculations** | Math is correct |
| **Inflation/interest** | Appropriate rates |
| **Starting balance** | Matches client data |

#### Professional Quality

| Item | Check |
|------|-------|
| **Spelling/grammar** | No errors |
| **Formatting** | Consistent throughout |
| **Photos** | Clear and relevant |
| **Branding** | Company info correct |

### QA Actions

| Action | When | Result |
|--------|------|--------|
| **Approve** | Report is ready | Advances to publishing |
| **Request Revisions** | Issues found | Returned to specialist with notes |

### Requesting Revisions

1. Click **"Request Revisions"**
2. **Specify issues** clearly
3. Reference **specific sections** or pages
4. Specialist receives notification
5. They address issues and resubmit

### Approving for Publishing

1. Complete your review
2. Click **"Approve"**
3. Status changes to **"ReportComplete"** or **"ApprovedReport"**
4. Study is ready for publishing

## Publishing Reports

The final step—delivering the report to the client.

### Pre-Publish Checklist

Before publishing:

- [ ] QA review completed
- [ ] Report approved
- [ ] Invoice status acceptable
- [ ] Client contact info correct
- [ ] Final PDF generated

### Publishing Process

1. Open the approved study
2. Click **"Publish"** or **"Publish to Client"**
3. Review what will be sent:
   - PDF report
   - Email notification
   - Portal access granted
4. Add optional message
5. Confirm publishing

### What Happens on Publish

1. **Report finalized** and timestamped
2. **Client notified** via email
3. **Portal access** granted for report
4. **Status** changes to **"RequestCompleted"**
5. **Invoice** may be auto-generated (final payment)

### Client Notification

The HOA client receives:

- 📧 **Email** announcing report completion
- 🔗 **Link** to view/download in portal
- 📄 **PDF attachment** (if configured)

## Amendments and Scope Changes

Sometimes the scope changes after work begins.

### What Triggers an Amendment

| Scenario | Example |
|----------|---------|
| **Additional components** | Discovered during site visit |
| **Removed components** | Not owned by HOA |
| **Significant variance** | Cost differs substantially |
| **Client request** | Wants to add/remove items |

### Amendment Workflow

```
Scope Change Detected → Amendment Created → Admin Reviews
                                                  ↓
                                        Approve → Send to Client
                                                  ↓
                                        Client Accepts → Continue Work
```

### Reviewing Amendments

When a scope change is flagged:

1. You receive notification
2. Review the **scope comparison**
3. See what was **added/removed**
4. Review **price impact**

### Amendment Actions

| Action | Result |
|--------|--------|
| **Approve** | Amendment sent to client |
| **Modify** | Adjust the amendment |
| **Reject** | Scope change denied |
| **Discuss** | Contact specialist for clarity |

### Client Amendment Acceptance

After you approve:

1. Client receives amendment
2. They review changes and new pricing
3. They accept or reject
4. If accepted, work continues
5. If rejected, discuss alternatives

## Troubleshooting

### Common Issues

**Q: A study is stuck in a status.**
A: Check if required actions are complete. Look for validation errors. May need specialist to fix data.

**Q: I approved but specialist didn't receive notification.**
A: Check specialist's email. Verify notification settings. Manually inform them.

**Q: Client hasn't responded to amendment.**
A: Follow up directly. Consider phone call. Resend if needed.

**Q: Report won't generate.**
A: Check for missing required fields. Verify all components have data. Contact support.

**Q: I published but client says they can't access.**
A: Verify client has active account. Check email delivery. Resend access link.
